Profiler yields per-function coverage, despite per-block level being set


If you enable coverage collection with per-block granularity, reloading page more than one or two times will cause coverage to have "per-function" level (per-block granularity is unavailable)

Issue is reproducible with both Chrome DevTools UI and Chrome DevTools Protocol. This example demonstrates the issue with Chrome DevTools UI
DevTools API Reference

Why is this an issue?

Q: Isn't reloading page once is sufficient to get a coverage? What is your use-case?
A: We are developing tool to collect coverage from UI automated tests (via DevTools Protocol API). Sometimes page gets reloaded hundreds of times. Coverage, from all steps, is important and block-level granularity is what really matters

Important notes:

  1. Issue is not reproducible on _short_ (see short.html) scripts
    This page loads long.js script, which is >10k LOC

  2. Issue is not reproducible when page is served from filesystem
    Page has to be served from webserver (If you cloned this repository, any webserver would fit: e.g. use SimpleHTTPServer if you have Python, or `npx serve` if you have Node.js installed)

  3. Issue is exactly reproducible with DevTools API (while Profiler.startPreciseCoverage({ detailed: true }) sets per-block level granularity, Profiler.takePreciseCoverage() yeilds per-function results (isBlockCoverage: false), if conditions described in Reproduction steps are met

  4. This issue does not appear, if you do hard-refresh each time you reload the page or disable the cache via DevTools completely
    . (This is extremely poor solution in case of tracking automated UI tests coverage, since modern webapps rely on cache)

The script:

The script is simple. Only "foo()" function matters, the rest is a bloat. Second branch in if statement should never be executed, and thus - covered, when coverage type is set to per-block

function foo(x) {
if (x) {
console.log('branch 1')
} else {
console.log('branch 2') // THIS LINE _CANNOT_ BE COVERED
}
}

foo(true);
// ...bloat

Reproduction steps:

  1. Serve _this_ page (index.html) with webserver of your choice (ignore, if you opened this from a website)
  2. Open it
  3. Open DevTools
  4. Open "Sources" tab on top, and select "long.js"
  5. Open "Coverage" tab on the bottom
  6. Set granularity level to "Per-block" and click "record"
  7. Refresh the page once
  8. Note the correct results: 1. in capture results coverage type has "per-block" value 2. Line marked with comment is not covered
  9. Refresh the page one more time
  10. Note that results are unchanged
  11. Refresh the page last time
  12. THE ISSUE:
    1. coverage type field has "per-function" value (despite "per-block" is set in settings)
    2. Line marked with comment is covered now (this is impossible when per-block coverage is set)
  13. If you hard-refresh (CTRL+F5) the page, then coverage has "per-block" granularity again

What is happening/my guess

It might be an issue with either how Chrome passes cached scripts to V8, or V8 itself handles cached/compiled scripts instrumentation (if such cache is a thing)
